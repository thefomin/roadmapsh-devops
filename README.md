# 📦 Log Archive Tool - Документация

## Обзор

Этот Bash-скрипт предназначен для автоматического архивирования логов на Linux-серверах. Он сжимает логи, сохраняет их в отдельную директорию и может отправлять уведомления через Telegram или Email, а также копировать архивы на удаленные серверы.

## Архитектура скрипта

### Основные компоненты

Скрипт состоит из нескольких функциональных блоков:

1. **Инициализация и конфигурация** - настройка переменных и загрузка окружения
2. **Загрузка .env файла** - безопасное чтение токенов и ключей
3. **Логирование** - запись всех действий в файл
4. **Создание архива** - сжатие логов с игнорированием ошибок
5. **Уведомления** - отправка оповещений (Email, Telegram)
6. **Удаленная отправка** - копирование архива на другие серверы
7. **Очистка** - удаление старых архивов

## Детальное описание работы

### 1. Конфигурация и цвета

```bash
RED='\033[0;31m'    # Красный цвет для ошибок
GREEN='\033[0;32m'  # Зеленый для успеха
YELLOW='\033[1;33m' # Желтый для предупреждений
BLUE='\033[0;34m'   # Синий для заголовков
NC='\033[0m'        # Сброс цвета

ARCHIVE_DIR="/var/log/archives"  # Куда сохранять архивы
LOG_FILE="/var/log/log-archive.log"  # Файл лога скрипта
DAYS_TO_KEEP=30  # Срок хранения архивов в днях

### 2. Загрузка переменных из .env

1. Ищет файл **.env** в директории скрипта
2. Читает перемнные в формате **КЛЮЧ=значение**
3. Игнорирует комментарие **(строки начинающиеся с #)**
4. Удаляет кавычки и лишние пробелы
5. Экспортирует переменные в окружение

**Формат .env файла:**
```bash
TELEGRAM_BOT_TOKEN="1234567890:ABCdefGHIjklmNOPqrstUVwxyz"
TELEGRAM_CHAT_ID="123456789"
``` 
### 3. Логирование

Функция **log_message():**

1. Добавляет временную метку к каждому сообщению
2. Записывает в файл и выводит в консоль
3. Использует уровни: INFO, WARNING, ERROR, SUCCESS

### 4. Создание архива

Функция **create_archive()** выполняет следующие шаги:

1. Генерация имени архива: **logs_archive_20260418_100756.tar.gz**
2. Проверка существования директории с логами
3. Создание временного файла исключений - чтобы не архивировать предыдущие архивы
4. Вызов tar с особыми параметрами:
``` --warning=no-file-changed ```  - игнорирует предупреждения об изменении файлов
```--ignore-failed-read```  - продолжает работу при ошибках чтения
```--exclude-from```  - исключает директорию архиво
```-czf``` - создать сжатый gzip архив
5. Фильтрация предупреждений - убирает сообщения об изменяющихся файлов и сокетах
6. Проверка результата - убеждается, что архив создан и не пуст
7. Возврат пути к архиу для дальнейшего использования

### 5. Отправка уведомлений

**Email уведомления(``send_email_notification``)**
1. Использует команду ``mail``
2. Отправляет детализированное письмо о создании архива
3. Требует настроенный почтовый сервер

**Telegram уведомления(``send_telegram_notification``)**
1. Использует Telegram Bot API через curl
2. Формирует сообщение с информацией об архиве
3. Отправляет POST запрос к ``https://api.telegram.org/bot<TOKEN>/sendMessage``
4. Проверяет успешность отправки по ответу API
